# Dify Flow - 内容润色工作流 (Content Polishing)

## 目标

对用户最终确认的文章内容进行语言层面的优化和润色，使其表达更流畅、自然，并结合基本的 SEO 原则进行调整，但 **绝对不能改变原文的核心意思和信息**。

## 工作流程

1.  **接收输入:** 获取用户确认后的最终文章文本。
2.  **文本分析:**
    *   分析句子结构、语法和用词。
    *   识别可能存在的不流畅、拗口或表达不清的地方。
3.  **语言润色与自动 SEO 微调:**
    *   进行同义词替换、语序调整、句式变换等操作，提升可读性和流畅性。
    *   确保语言风格统一。
    *   LLM 会自动分析文章内容，识别核心主题和潜在关键词，并在不改变原文核心意思、不影响自然流畅的前提下，进行轻微的、自然的表达调整，以优化 SEO 友好度。
    *   **约束:** 严禁生硬堆砌关键词或为了 SEO 而牺牲内容质量和可读性。所有 SEO 相关的改动都必须以不影响原文核心内容和观点为最高优先级。
4.  **输出结果:** 返回经过润色和轻微 SEO 优化的文本内容。

## 输入参数

*   `article_content` (string): 用户确认的最终文章内容。

## 输出结果

*   `polished_content` (string): 经过润色和轻微 SEO 优化的文章内容。

## 关键约束

*   **内容保真:** 润色过程严格遵守不改变原文核心意思、信息和观点的原则。
*   **用户确认:** 润色后的内容仍需用户最终确认。

## LLM 节点 Prompts 示例

以下是在 Dify 工作流中配置 LLM 节点时可以使用的 Prompt 示例：

### System Prompt

```text
你是一位经验丰富的中文编辑和 SEO 专家。你的任务是润色用户提供的文章内容，使其语言表达更流畅、自然、专业。同时，你需要自动分析文章内容，识别核心主题和关键词，并在不改变核心内容、不影响阅读体验的前提下，进行自然的 SEO 微调。请严格遵守以下规则：

1.  **核心内容不变**：绝对不允许修改文章的核心观点、事实信息或数据。润色仅限于语言表达层面。
2.  **保持风格**：尽量保持原文的整体语气和风格。
3.  **流畅性优先**：优化语句结构，消除语病和拗口的表达。
4.  **自动 SEO 微调**：在润色过程中，自然地优化表达，使其更符合 SEO 习惯。识别并围绕核心主题/关键词进行微调，但必须保证：
    *   不生硬添加或堆砌关键词。
    *   不改变原文核心意思和信息。
    *   调整后的语句依然自然流畅，符合中文表达习惯。
    *   优先保证内容质量和可读性，SEO 优化是次要目标。
5.  **直接输出润色后的全文**：不要添加任何解释性文字或评论，直接输出润色后的完整文章内容。
```
